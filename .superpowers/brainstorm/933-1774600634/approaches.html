3 Architectural Approaches

How should the new social system be structured? Pick the one that fits how you work.

A

Platform-First Queue

Each platform is its own lane. Generate posts per-platform, edit individually, schedule each one. Image/video generation is triggered per post slot when needed.

Pros

  • Maximum per-platform control
  • Easy to override one platform without touching others

Cons

  • More clicks per article (6 platforms = 6 separate workflows)
  • Doesn't match Ava's batch approach
B

Content Kit (Recommended)

One article → one "Content Kit" generated in a single batch: all platform captions + image prompts + Reel script + auto-suggested schedule. Review the kit, approve it, it deploys itself across the week.

Matches Ava's methodology: batch content creation, Instagram-first with cross-post cascade, optimal timing built in.

Pros

  • One approval → all 6 platforms scheduled
  • Content pillar auto-tagged (Educate/Entertain/Sell)
  • Ava's hook archetypes baked into generation
  • Minimal friction — matches existing article-centric UI

Cons

  • Less granular per-platform control at generation time (edit after)
C

Campaign Builder

Decouple generation from articles. Build standalone social campaigns — pick a topic, choose pillar (Educate/Entertain/Sell), generate a month's worth of content at once using Ava's calendar framework.

Pros

  • Most powerful for pure social strategy
  • Full Ava 4-week calendar framework
  • Content independent of article pipeline

Cons

  • Biggest rebuild — doesn't reuse existing article flow
  • More complex to learn and operate
  • SLAHEALTH content is article-driven — this fights that
All three approaches include: Ava's HOOK/BODY/CTA generation, content pillar tagging, image generation via OpenRouter (Gemini Flash / Riverflow), video via FAL.ai (Kling 9:16), and API posting to all 6 platforms.
